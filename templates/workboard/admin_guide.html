Supervisor guide

A short walkthrough for the parts of TaskForge supervisors use most often.

Daily workflow

  1. Check Board for overdue, waiting, and today's scheduled work.
  2. Use Intake when someone sends a freeform request that needs parsing.
  3. Use New Task for tasks you already understand and want to enter manually.
  4. Open Reports when you need a quick weekly workload summary.

Roles

Supervisor
Can create tasks, manage people, schedules, recurring work, reports, and passwords.
Student Supervisor
Can see the whole board and edit tasks, but cannot create new tasks or manage people.
Student Worker
Can see assigned work, update status, and add notes or attachments.

Recurring tasks

The Recurring page shows the templates that generate repeated work. Open a recurring task to:

  • See the next run preview and upcoming run dates.
  • Confirm whether the next run will stay with a fixed assignee or rotate to a different worker.
  • Use Run now when you need the next recurring task to appear immediately.

TaskForge will not run a recurring task again while the current run is still open.

Schedules and temporary changes

On the Workers page, use Edit schedule for the weekly calendar.

  • Drag across the weekly calendar to add blocks in 30-minute increments.
  • Use the weekday copy buttons to copy one day across the rest of the week.
  • Use Temporary schedule change for a one-date exception such as time off or a special shift.
  • The Load normal schedule button pulls that weekday's normal schedule into the one-date override so you can adjust it instead of rebuilding it from scratch.

A temporary schedule overrides the weekly schedule only for that date.

People management

  • Use Edit worker or Edit supervisor to update names, email, notes, and role-specific settings.
  • Use Reset password when someone is locked out or needs a fresh login.
  • Removing a person reassigns their open work to the acting supervisor so tasks do not get stranded.

Testing and checks

Useful commands from the project root:

python manage.py check
python manage.py test workboard.tests
python manage.py generate_recurring_tasks

Use the full test suite before pushing larger workflow changes, especially anything involving recurring tasks, schedules, or permissions.

Troubleshooting

  • If a recurring task is missing from the Recurring page, open the page once to let TaskForge backfill older recurring-marked tasks into templates.
  • If a worker cannot be assigned to a scheduled task, check both their weekly schedule and any temporary schedule for that date.
  • If the board looks stale after a deploy, do a hard refresh so the newest CSS and JavaScript files load.
